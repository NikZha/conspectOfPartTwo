Опережающие и ретроспективные проверки

В некоторых случаях нам нужно найти соответствия шаблону, но только те, за которыми или перед которыми следует другой шаблон. Для этого в регулярных выражениях есть специальный синтаксис: опережающая (lookahead) и ретроспективная (lookbehind) проверка.

Опережающая проверка

Бывает как на соотвествие одному условию X(?=Y), X - это искомое если за ним следует Y. Вместо X и Y может быть любой шаблон. Например:


let str = "1 индейка стоит 30€";

alert( str.match(/\d+(?=€)/) ); // 30, число 1 проигнорировано, так как за ним НЕ следует €

Более сложный шаблон X(?=Y)(?=Z) проверяет на удовлетворение двум условиям сразу: Y и Z и если что-то не совпадает - регексп не работает:


let str = "1 индейка стоит 30€";

alert( str.match(/\d+(?=\s)(?=.*30)/) ); // 1

Негативная опережающая проверка

Синтаксис: X(?!Y) Он означает: найди такой X, за которым НЕ следует Y: 


let str = "2 индейки стоят 60€";

alert( str.match(/\d+(?!€)/) ); // 2 (в этот раз проигнорирована цена)

Ретроспективная проверка

Бывает позитивная, а бывает негативная проверки. Синтаксис:

  • Позитивная ретроспективная проверка: (?<=Y)X, ищет совпадение с X при условии, что перед ним ЕСТЬ Y. 
    
let str = "1 индейка стоит $30";

// знак доллара экранируем \$, так как это специальный символ
alert( str.match(/(?<=\$)\d+/) ); // 30, одинокое число игнорируется
  • Негативная ретроспективная проверка: (?<!Y)X, ищет совпадение с X при условии, что перед ним НЕТ Y. 
    
let str = "2 индейки стоят $60";

alert( str.match(/(?<!\$)\d+/) ); // 2 (проигнорировалась цена)

Скобочные группы

В позитивных проверках можно использовать скобочную группу для включения того что находится в скобочной группе в результат.

Опережающая проверка:


let str = "1 индейка стоит 30€";
let regexp = /\d+(?=(€|kr))/; // добавлены дополнительные скобки вокруг €|kr

alert( str.match(regexp) ); // 30, €

Ретроспективная проверка:


let str = "1 индейка стоит $30";
let regexp = /(?<=(\$|£))\d+/;

alert( str.match(regexp) ); // 30, $

Как видно из примеров достигается это заключением поискового шаблона в (€|kr) скобочную группу.

Итого

Виды проверок:

шаблон Тип Совпадение
X(?=Y) Позитивная опережающая X, если за ним следует Y
X(?!Y) Негативная опережающая X, если за ним НЕ следует Y
(?<=Y)X Позитивная ретроспективная X, если следует за Y
(?<!Y)X Негативная ретроспективная X, если НЕ следует за Y
