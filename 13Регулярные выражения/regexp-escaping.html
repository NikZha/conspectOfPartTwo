Экранирование, специальные символы

Помимо \ есть и другие управляющие символы: [ ] \ ^ $ . | ? * + ( )

Экранирование символов

Экранирование символов осуществляется косой чертой \, что позволяет искать в тексте специфические символы:


alert( "Глава 5.1".match(/\d\.\d/) ); // 5.1 (совпадение!)
alert( "Глава 511".match(/\d\.\d/) ); // null ("\." - ищет обычную точку)
alert( "function g()".match(/g\(\)/) ); // "g()"
alert( "1\\2".match(/\\/) ); // '\'

Косая черта (slash)

Косая черта тоже экранируется, так как использутся в самом языке для создания регекспов /.../, или используются new RegExp('/'): 


alert( "/".match(/\//) ); // '/'
alert( "/".match(new RegExp("/")) ); // находит /

new RegExp

Как я столкнулся в задаче - \ обратный слеш в строках надо экранировать:


let regStr = "\\d\\.\\d";
alert(regStr); // \d\.\d (теперь правильно)

let regexp = new RegExp(regStr);

alert( "Глава 5.1".match(regexp) ); // 5.1

Итого

  • Для поиска специальных символов [ ] \ ^ $ . | ? * + ( ), нам нужно добавить перед ними \ («экранировать их»).
  • Нам также нужно экранировать /, если мы используем в /.../ (но не new RegExp).
  • При передаче строки в new RegExp нужно удваивать обратную косую черту: \\ для экранирования специальных символов, потому что строковые кавычки «съедят» одну черту.
