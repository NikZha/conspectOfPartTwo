FormData


let formData = new FormData([form]);            

Это конструктор создающий соотвествующий типа даных, если передаётся в этот конструктор html-элемент form, то прочитываются её поля.


При обмене данными с сетью указывается тип данных multipart/form-data как и в случае обычной форме в запросе  POST.

Отправка простой формы

Создание простой формы:


        <form id="formElem">
            <input type="text" name="name" value="John">
            <input type="text" name="surname" value="Smith">
            <input type="submit">
        </form>

        <script>
            formElem.onsubmit = async (e) => {
                e.preventDefault();

                let response = await fetch('/article/formdata/post/user', {
                    method: 'POST',
                    body: new FormData(formElem)
                });

                let result = await response.json();

                alert(result.message);
            };
        </script>

Пример у меня не работает, потому что нет серверной части которая принимает эту форму и отвечает на запрос.

Но в целом понятно, что действие по умолчанию отменяется, а сама форма создаётся в виде объекта FromData и отправляется при помощи fetch, а далее читается ответ сервера.

Методы объекта FormData

Список методов похожий на Map:

  • formData.append(name, value) – добавляет к объекту поле с именем name и значением value,
  • formData.append(name, blob, fileName) – добавляет поле, как будто в форме имеется элемент <input type="file">, третий аргумент fileName устанавливает имя файла (не имя поля формы), как будто это имя из файловой системы пользователя,
  • formData.set(name, value),
  • formData.set(name, blob, fileName).
  • formData.delete(name) – удаляет поле с заданным именем name,
  • formData.get(name) – получает значение поля с именем name,
  • formData.has(name) – если существует поле с именем name, то возвращает true, иначе false

Так же как и Map FormData может итерироваться по key, value:


let formData = new FormData();
formData.append('name', 'Иван');
formData.append('surname', 'Иванов');

// Список пар ключ/значение
for(let [name, value] of formData) {
  alert(`${name} = ${value}`); // key1=value1, потом key2=value2
}

Отправка формы с файлом

Собственно все тоже самое кроме того, что помимо текстовых полей в форме присуствиуем <input type="file"> потому что заголовок Content-Type: multipart/form-data позволяет отправлять файлы.

У автора этот пример не работает, нет серверного механизма обрабатывающего запрос:


<form id="formElem">
  <input type="text" name="firstName" value="John">
  Картинка: <input type="file" name="picture" accept="image/*">
  <input type="submit">
</form>

<script>
  formElem.onsubmit = async (e) => {
    e.preventDefault();

    let response = await fetch('/article/formdata/post/user-avatar', {
      method: 'POST',
      body: new FormData(formElem)
    });

    let result = await response.json();

    alert(result.message);
  };
</script>

Отправка формы с Blob-данными

В топике приводится длинный неработающий пример с canvas, но основное это:


formData.append("image", imageBlob, "image.png");

То есть прямо указывается, что это image, добавляется переменная содержащая blob и третий аргумент это имя файла. Далее сервер считает, что это обычный файл из формы.
