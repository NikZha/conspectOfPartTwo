LocalStorage, sessionStorage

Оба этих объекта позволяют хранить ключ-значения в браузере. Чем похожи на Map. С тем отличием от кук и Map что:

  • Не отправляют в каждом запросе как заголовок.
  • Так же объем может быть значительно больше - от 5 мб и далее.
  • Веб-сервер не может манипулировать этими объектами. Только javascript.
  • Хранилице соотвествует - протокол://домен:порт и другое сочетание не может получить доступ к хранилищу.

Методы этих объектов:

  • setItem(key, value) – сохранить пару ключ/значение.
  • getItem(key) – получить данные по ключу key.
  • removeItem(key) – удалить данные с ключом key.
  • clear() – удалить всё.
  • key(index) – получить ключ на заданной позиции.
  • length – количество элементов в хранилище.

Интерфейс сильно похож на Map, однако есть key(index) – получить ключ на заданной позиции.

Демо localStorage

Основные особенности localStorage:

  • Этот объект один на все вкладки и окна в рамках источника (один и тот же домен/протокол/порт).
  • Данные не имеют срока давности, по которому истекают и удаляются. Сохраняются после перезапуска браузера и даже ОС.

Демо:


localStorage.setItem('test', 1);
alert( localStorage.getItem('test') ); // 1

Срабатывает и на локальном сервере и будучи открытым с диска.

Доступ как к обычному объекту

Можно получать доступ к хранилищу, как к обычному объекту, оставлено это для совместимости:


// установить значение для ключа
localStorage.test = 2;

// получить значение по ключу
alert( localStorage.test ); // 2

// удалить ключ
delete localStorage.test;

Но лучше так не делать, потому что:

  • Ключ может не проканать. Всё-таки это объект и все свойства взять и установить может и не получится.
  • Только при модификации хранилища при помощи setItem срабатывает событие storage.

Перебор ключей


//Как по "обычному" массиву:
for(let i=0; i<localStorage.length; i++) {
  let key = localStorage.key(i);
  alert(`${key}: ${localStorage.getItem(key)}`);
}
// bad try
for(let key in localStorage) {
  alert(key); // покажет getItem, setItem и другие встроенные свойства
}
//Только те ключи, которые не являются изначальными ключами объекта
for(let key in localStorage) {
  if (!localStorage.hasOwnProperty(key)) {
    continue; // пропустит такие ключи, как "setItem", "getItem" и так далее
  }
  alert(`${key}: ${localStorage.getItem(key)}`);
} 
//получаем массив ключей: 
let keys = Object.keys(localStorage);
for(let key of keys) {
  alert(`${key}: ${localStorage.getItem(key)}`);
}
