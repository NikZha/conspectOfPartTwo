Атака типа clickjacking

Захват клика и авторизации на стороннем сайте. Пишет что сайты типа Twitter, Facebook, Paypal сейчас защищены.

Идея

Взлом жопы фейсбука жертвы при помощи:

  1. Посетителя заманивают на вредоносную страницу (неважно как).
  2. На странице есть ссылка, которая выглядит безобидно (например, «Разбогатей прямо сейчас» или «Нажми здесь, это очень смешно»).
  3. Поверх этой ссылки вредоносная страница размещает прозрачный <iframe> с src с сайта facebook.com таким образом, что кнопка «like» находится прямо над этой ссылкой. Обычно это делается с помощью z-index в CSS.
  4. При попытке клика на эту ссылку посетитель на самом деле нажимает на кнопку.

Демонстрация


<style>
iframe { /* ифрейм с сайта-жертвы */
  width: 400px;
  height: 100px;
  position: absolute;
  top:0; left:-20px;
  opacity: 0.5; /* в реальности opacity:0 */
  z-index: 1;
}
</style>

<div>Нажми, чтобы разбогатеть:</div>

<!-- Url с сайта-жертвы -->
<iframe src="/clickjacking/facebook.html"></iframe>

<button>Нажмите сюда!</button>

Короче идея постая, распологаем кнопку авторизации на сайте жертвы в фрейме при помощи css перед своей, жертва кликает и отдаёт свою авторизацию злоумышленнику.

Clickjacking-атака для кликов мыши, а не для клавиатуры

Плюс-минус тоже самое можно провернуть и для клавиатуры, но проблема в том, что получить клавиатурый ввод со стороннего сайта невозможно, а пользователь не будет продолжать что-то вводить не видя ввод.
