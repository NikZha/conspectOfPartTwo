LocalStorage, sessionStorage

Оба этих объекта позволяют хранить ключ-значения в браузере. Чем похожи на Map. С тем отличием от кук и Map что:

  • Не отправляют в каждом запросе как заголовок.
  • Так же объем может быть значительно больше - от 5 мб и далее.
  • Веб-сервер не может манипулировать этими объектами. Только javascript.
  • Хранилице соотвествует - протокол://домен:порт и другое сочетание не может получить доступ к хранилищу.

Методы этих объектов:

  • setItem(key, value) – сохранить пару ключ/значение.
  • getItem(key) – получить данные по ключу key.
  • removeItem(key) – удалить данные с ключом key.
  • clear() – удалить всё.
  • key(index) – получить ключ на заданной позиции.
  • length – количество элементов в хранилище.

Интерфейс сильно похож на Map, однако есть key(index) – получить ключ на заданной позиции.

Демо localStorage

Основные особенности localStorage:

  • Этот объект один на все вкладки и окна в рамках источника (один и тот же домен/протокол/порт).
  • Данные не имеют срока давности, по которому истекают и удаляются. Сохраняются после перезапуска браузера и даже ОС.

Демо:


localStorage.setItem('test', 1);
alert( localStorage.getItem('test') ); // 1

Срабатывает и на локальном сервере и будучи открытым с диска.

Доступ как к обычному объекту

Можно получать доступ к хранилищу, как к обычному объекту, оставлено это для совместимости:


// установить значение для ключа
localStorage.test = 2;

// получить значение по ключу
alert( localStorage.test ); // 2

// удалить ключ
delete localStorage.test;

Но лучше так не делать, потому что:

  • Ключ может не проканать. Всё-таки это объект и все свойства взять и установить может и не получится.
  • Только при модификации хранилища при помощи setItem срабатывает событие storage.

Перебор ключей


//Как по "обычному" массиву:
for(let i=0; i<localStorage.length; i++) {
  let key = localStorage.key(i);
  alert(`${key}: ${localStorage.getItem(key)}`);
}
// bad try
for(let key in localStorage) {
  alert(key); // покажет getItem, setItem и другие встроенные свойства
}
//Только те ключи, которые не являются изначальными ключами объекта
for(let key in localStorage) {
  if (!localStorage.hasOwnProperty(key)) {
    continue; // пропустит такие ключи, как "setItem", "getItem" и так далее
  }
  alert(`${key}: ${localStorage.getItem(key)}`);
} 
//получаем массив ключей: 
let keys = Object.keys(localStorage);
for(let key of keys) {
  alert(`${key}: ${localStorage.getItem(key)}`);
}

Только строки

Обратите внимание, что ключ и значение должны быть строками. Далее идёт пример, в котором объект превращается в toString() - [object Object] и таким образом теряется его внутреннее содержание:


                 
localStorage.user = {name: "John"};
alert(localStorage.user); // [object Object]

Избежать этого поможет JSON:


localStorage.user = JSON.stringify({name: "John"});

// немного позже
let user = JSON.parse( localStorage.user );
alert( user.name ); // John

Также возможно привести к строке весь объект хранилища, например для отладки:


// для JSON.stringify добавлены параметры форматирования, чтобы объект выглядел лучше
alert( JSON.stringify(localStorage, null, 2) );

sessionStorage

Особенности sessionStorage:

  • sessionStorage существует только в рамках текущей вкладки браузера.
    • Другая вкладка с той же страницей будет иметь другое хранилище.
    • Но оно разделяется между ифреймами на той же вкладке (при условии, что они из одного и того же источника).
  • Данные продолжают существовать после перезагрузки страницы, но не после закрытия/открытия вкладки.

Методы те же самые.

Событие storage

При изменении какого-либо хранилища генерирутся событие-объект storage со следующими свойствами:

  • key – ключ, который обновился (null, если вызван .clear()).
  • oldValue – старое значение (null, если ключ добавлен впервые).
  • newValue – новое значение (null, если ключ был удалён).
  • url – url документа, где произошло обновление.
  • storageArea – объект localStorage или sessionStorage, где произошло обновление.

Это код работает так, что в одном объекте добавляется что-либо в другом срабатывает событие.


// срабатывает при обновлениях, сделанных в том же хранилище из других документов
window.onstorage = event => { // можно также использовать window.addEventListener('storage', event => {
  if (event.key != 'now') return;
  alert(event.key + ':' + event.newValue + " at " + event.url);
};

localStorage.setItem('now', Date.now());

Сделано исключительно для обмена данными между вкладками с одного ресурса.
