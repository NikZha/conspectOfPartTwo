Квантификаторы +, *, ? и {n}

Квантификаторы это мультипликаторы для классов \w, \d и так далее.

Количество {n}

Самый простой квантификатор — это число в фигурных скобках: {n}.Может быть разным:

  • Точное количество: {5}

    Шаблон \d{5} обозначает ровно 5 цифр, он эквивалентен \d\d\d\d\d.

    
alert( "Мне 12345 лет".match(/\d{5}/) ); //  "12345"

    Шаблон может быть длиннее: \b\d{5}\b.

  • Диапазон: {3,5}, от 3 до 5

    Можно указывать диапазон поиска \d{3,5}

    
alert( "Мне не 12, а 1234 года".match(/\d{2,5}/g) ); // "12,1234"

    Так же можно не указывать верхнюю границу поиска: \d{1,}:

    
let str = "+7(903)-123-45-67";

let numbers = str.match(/\d{1,}/g);

alert(numbers); // 7,903,123,45,67

Короткие обозначения

Для самых востребованных квантификаторов есть сокращённые формы записи:

  • +
    Означает «один или более». То же самое, что и {1,}. Например, \d+ находит числа (из одной или более цифр): 
    
let str = "+7(903)-123-45-67";

alert( str.match(/\d+/g) ); // 7,903,123,45,67
  • ?
    Означает «ноль или один». То же самое, что и {0,1}. По сути, делает символ необязательным. 
    
let str = "Следует писать color или colour?";

alert( str.match(/colou?r/g) ); // color, colour

    То есть поскольку символ необязательный найдёт с ним и без него.

  • *
    Означает «ноль или более». То же самое, что и {0,}. 
    
alert( "100 10 1".match(/\d0*/g) ); // 100, 10, 1

    Тут смысл в том, что это * относится к 0 и если сравнивать с \d0+, то единицу оно не найдёт поскольку там шаблон {1,}.

Ещё примеры

  • Регулярное выражение для десятичных дробей (чисел с плавающей точкой): \d+\.\d+ 
    
alert( "0 1 12.345 7890".match(/\d+\.\d+/g) ); // 12.345
  • Регулярное выражение для «открывающего HTML-тега без атрибутов», например, <span> или <p> 
    
alert( "<body> ...</body>".match(/<[a-z]+>/gi) ); // <body>
  • Регулярное выражение для «открывающего или закрывающего HTML-тега без атрибутов»: /<\/?[a-z][a-z0-9]*>/i 
    
alert( "<h1>Привет!</h1>".match(/<\/?[a-z][a-z0-9]*>/gi) ); // <h1>, </h1>
