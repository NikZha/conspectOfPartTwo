Скрипты: async, defer

Основная проблема скриптов, в том, что они:

  1. Блокируют загрузку контента ниже.
  2. "Не видят" контент ниже.

Это та самая проблема с которой я столкнулся на выходных когда писал скрипт наскрашивающий отдельно стоящие блоки <code> получая просто странные мистические ошибки. Решением этой проблемы и являются астрибуты defer и async в скриптах.

defer

Атрибут дефер сообщает браузеру, чтобы не ждал загрузку скрипта, а выполнял бы загрузку документа полностью и только потом запускал бы скрипт. Иллюстрацией этого является скрипт из этого документа:


<script defer src="../scripts.js"></script>

Который устанавливает класс отдельно стоящим блокам code, а так же:

  • Скрипты с defer никогда не блокируют страницу.
  • Скрипты с defer всегда выполняются, когда дерево DOM готово, но до события DOMContentLoaded.

Скрипты с defer выполняются в порядке расположения на странице, даже если загрузится скрипт который стоит позже, выполняться сначала будет скрипт который стоит раньше. Что бывает критически важным при загрузке библиотек и кода их используемых.

Атрибут defer предназначен только для внешних скриптов

Для скрипов на странице атрибут defer будет проигнорирован.

async

Атрибут async означает, что скрипт полностью независим от загрузки кантента на странице, от загрузки и выполнения других скриптов. Нет никаких гарантий относительно времени выполнения такого скрипта и события DOMContentLoaded

Асинхронные скрипты очень полезны для добавления на страницу сторонних скриптов: счётчиков, рекламы и т.д. Они не зависят от наших скриптов, и мы тоже не должны ждать их:


<!-- Типичное подключение скрипта Google Analytics -->
<script async src="https://google-analytics.com/analytics.js"></script>

Атрибут async предназначен только для внешних скриптов

Как и в случае с defer, атрибут async будет проигнорирован, если в теге <script> нет src.

Динамически загружаемые скрипты

Скрипты можно добавлять на страницу динамически из кода других скриптов:


let script = document.createElement('script');
script.src = "/article/script-async-defer/long.js";
document.body.append(script); // (*)
  1. Скрипт начнёт загружаться как только будет добавлен на страницу (*)
  2. По умолчанию такой скрипт является async

Убрать async можно при помощи script.async = false: let script = document.createElement('script'); script.src = "/article/script-async-defer/long.js"; script.async = false; document.body.append(script);

Итого

На практике defer используется для скриптов, которым требуется доступ ко всему DOM и/или важен их относительный порядок выполнения.

А async хорош для независимых скриптов, например счётчиков и рекламы, относительный порядок выполнения которых не играет роли.
